# WeChat DevTool Changelog

## [重构偏好设置] - {PR_MERGE_DATE}

### 优化

- 重构偏好设置结构，移除冗余配置项
- 将设备名称和 CLI 路径配置移至图形化界面
- 新增"显示通知"选项，可控制操作反馈
- 新增"默认回退设备"选项，支持更灵活的设备回退策略
- 优化配置管理逻辑，提升用户体验

### 变更

- 偏好设置现在只包含全局选项，设备配置完全通过图形化界面管理
- 设备回退策略支持"使用默认设备"和"显示错误"两种模式
- 简化了配置流程，减少用户困惑

## [多设备配置支持] - {PR_MERGE_DATE}

### 新增

- 支持多设备独立配置，每台设备有独立的 CLI 路径和项目列表
- 自动设备检测，使用 `scutil --get ComputerName` 获取设备名
- 设备配置回退机制，未匹配设备时使用默认配置
- 完全图形化配置界面，支持动态添加设备和项目
- 项目 UUID 自动生成，无需手动管理

### 功能

- 设备管理：添加、编辑、删除设备配置
- 项目管理：为每个设备独立管理项目列表
- 配置持久化：使用本地 JSON 文件存储配置
- 界面优化：显示当前设备和配置状态

## [初始版本] - {PR_MERGE_DATE}

### 新增

- 基础微信开发者工具 CLI 集成
- 项目列表显示和搜索功能
- 通过 CLI 打开项目功能
- 成功/失败操作反馈
- 基础配置管理
